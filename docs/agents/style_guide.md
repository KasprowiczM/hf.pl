# Style Guide

## Purpose
Keep code small, readable, and stable for a marketing site with clear content intent.

## Rules
- Prefer simple React components over abstraction-heavy patterns.
- Keep functions short; extract helpers when logic branches multiply.
- Use explicit names for props and state; avoid single-letter variables.
- Keep comments rare and only where intent is non-obvious.
- Preserve existing i18n key structure; do not hardcode bilingual copy in JSX.
